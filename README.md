# 🛡 **SOC & Threat Monitoring Lab**

---

## 📌 **Project Overview**

This repository showcases hands-on practice in a Security Operations Center (SOC) environment, focusing on threat detection, log analysis, network monitoring, and incident handling.

The lab simulates real-world attack scenarios such as brute-force attempts, port scanning, and web-based activities, helping build practical understanding of security monitoring workflows.

---

## 🎯 **Objectives**

* Monitor and analyze security events in a controlled lab setup
* Detect suspicious activities using log correlation techniques
* Perform packet-level traffic inspection
* Classify and document security incidents
* Understand the basic SOC incident response process

---

## 🛠 **Tools Used**

* **Splunk** – Log analysis and monitoring (SIEM-based approach)
* **Wireshark** – Network packet inspection
* **Kali Linux** – Attack simulation environment
* **Nmap** – Port scanning and reconnaissance
* **Metasploit** – Exploitation and testing

---

## 🔍 **Activities Performed**

### 1️⃣ Log Monitoring & Analysis

* Ingested logs into Splunk for monitoring simulated attack activity
* Created SPL queries to identify unusual patterns
* Analyzed failed authentication attempts and abnormal login behavior
* Correlated multiple events to detect brute-force activity

---

### 2️⃣ Network Traffic Analysis

* Captured live traffic using Wireshark
* Inspected packets for unusual source and destination behavior
* Identified scanning attempts and irregular protocol usage

---

### 3️⃣ Threat Detection Scenarios

* Brute-force attack detection
* Port scanning identification
* Web activity monitoring for suspicious behavior
* Analysis of abnormal traffic patterns

---

### 4️⃣ Incident Classification

* Categorized incidents based on severity (Low / Medium / High)
* Documented findings in structured format
* Followed a basic incident response workflow:

  * Identification
  * Analysis
  * Containment
  * Documentation

---

## 📊 **Key Learning Outcomes**

* Understanding of SOC monitoring workflow
* Hands-on experience with SIEM-style log analysis
* Improved ability to detect anomalies using log correlation
* Practical exposure to network traffic analysis
* Better incident documentation and reporting skills
